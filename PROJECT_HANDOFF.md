# goai.rest — Полное резюме проекта для продолжения в Cowork

## 1. О проекте

**Название:** goai.rest — Restaurant Analytics Dashboard  
**Цель:** Единый дашборд аналитики для сети столовых/ресторанов МГУ (31 точка). Подключается к R-Keeper 7 (SQL Server) и StoreHouse (REST API), показывает выручку, фудкост, P&L, прогнозы, ИИ-рекомендации.  
**Текущая версия:** v9.41  
**GitHub:** https://github.com/wentur/analytics  
**Хостинг:** Streamlit Cloud или собственный сервер  
**Язык интерфейса:** Русский + Английский (переключатель в сайдбаре и на логине)

---

## 2. Стек и технологии

| Компонент | Технология |
|---|---|
| **Frontend/Backend** | Streamlit (Python, один файл `dashboard.py`, ~9700 строк) |
| **Графики** | Plotly (express + graph_objects), 109 графиков |
| **БД продаж** | R-Keeper 7 SQL Server (pymssql) |
| **БД склада** | StoreHouse 5 REST API (requests) |
| **ИИ-чат** | Google Gemini API |
| **Настройки пользователей** | SQLite (файл `goai_settings.db`, создаётся автоматически) |
| **Демо-режим** | SQLite in-memory из `demo_data.json.gz` |
| **Аутентификация** | Хешированные пароли в коде + URL-токены |
| **Мультиязычность** | Словарь `_TRANSLATIONS` + функции `t()`, `tp()`, `tg()` |

## 5. Архитектура dashboard.py

Файл монолитный (~9700 строк). Основные секции:

### Верхний уровень (строки 1–500)
- **Авто-настройка** `config.toml` → `.streamlit/config.toml`
- **Загрузка `.env`** — поддержка и `.env`, и `env` (без точки)
- **Импорты** — streamlit, pandas, numpy, plotly, pymssql, requests
- **Конфигурация** — `DB_CONFIG`, `SH_CONFIG`, `GEMINI_API_KEY`
- **Аутентификация** — `AUTH_USERS`, `_verify_user()`, `check_auth()`, URL-токены
- **Настройки пользователей** — SQLite (`save_user_setting`, `load_user_setting`)
- **Мультиязычность** — `_TRANSLATIONS` dict (~340 ключей), `t()`, `tp()`, `tg()`, `_period_key_to_label()`, `_period_label_to_key()`
- **Логин-страница** — `show_login_page()` с переключателем языка 🇷🇺/🇬🇧

### CSS и тема (строки 500–1600)
- **Полная кастомная тема** — CSS-переменные `--card`, `--border`, `--green`, `--t1..t3`
- **Светлая/тёмная тема** — переключатель в сайдбаре, сохраняется в URL `?theme=light`
- **Мобильный CSS** — `@media (max-width: 768px)` с flex-wrap, 2-колоночной сеткой
- **JS-фиксы** — MutationObserver для: статус-виджета, автозакрытия сайдбара, периода

### Подключение к данным (строки 1600–2500)
- **`get_connection()`** — pymssql к RK7 с retry
- **`run_query_cached()`** — обёртка с `@st.cache_data(ttl=300)`
- **`sh_api_call()`** — запросы к StoreHouse API
- **`sh5exec()`** — выполнение SH-процедур через `/api/sh5exec`
- **Демо-режим** — SQLite in-memory из `demo_data.json.gz`

### Функции загрузки данных (строки 2500–4200)
- `load_orders()`, `load_hourly()`, `load_daily()` — заказы из RK7
- `load_restaurants()`, `load_categories()`, `load_dishes()` — справочники
- `load_voids()`, `load_payments()`, `load_check_count()` — чеки/оплаты
- `load_tax_breakdown()` — выручка по ставкам НДС
- `load_abc_analysis()` — ABC-анализ блюд
- `sh_load_purchase_prices()` — закупочные цены из SH-накладных
- `sh_load_recipe_foodcost()` — себестоимость из рецептур (GDoc12)
- `match_foodcost()` — сопоставление RK ↔ SH (точное + подстрока + токенное)
- `_tokenize_name()`, `_token_match_score()` — токенное сопоставление названий
- `_staff_meal_cost()` — себестоимость питания сотрудников
- `_fixed_cost_for_period()` — постоянные расходы по календарным месяцам

### Тема и навигация (строки 4200–4500)
- **`CHART_THEME`** — dict для Plotly (light/dark), вкл. `dragmode=False`
- **`_labels()`** — двуязычные подписи осей для Plotly (функция, не dict)
- **`MENU_GROUPS`** — структура навигации
- **`page_header()`** — заголовок страницы + период-селектор
- **`period_selector()`** — компактный `[◀][Период][▶][↻]` с разным layout для пресетов и произвольного

## 7. Ключевые решения и подходы

### 7.1 Монолитный файл
Весь дашборд — один файл `dashboard.py`. Причина: Streamlit Cloud и простота деплоя. Минус: файл уже ~9700 строк, сложно навигировать.

### 7.2 Мультиязычность
- Словарь `_TRANSLATIONS` с ключами `"ru"` и `"en"` (~340 ключей)
- Функция `t("key")` → возвращает перевод по текущему языку
- `tp("Выручка")` → "Revenue" (перевод названия страницы)
- `tg("Продажи")` → "Sales" (перевод группы навигации)
- `_labels()` → dict для Plotly axes (функция, вызывается каждый рендер)
- `_period_key_to_label()` / `_period_label_to_key()` — перевод пресетов периода
- Язык хранится в `st.session_state["_lang"]`, переключатель на логине и в сайдбаре

### 7.3 Себестоимость питания сотрудников
- Сотрудники едят по PAYLINETYPE=3
- **Вычитается себестоимость, а не цена по меню** (500₽ по меню × 33% фудкост = 165₽ реальный расход)
- Если фудкост рассчитан → точный %, иначе → 33% (оценка)
- Функция `_staff_meal_cost(revenue)` → `(cost, fc_pct, is_exact)`

### 7.4 Фудкост — сопоставление товаров
- R-Keeper и StoreHouse используют разные названия для одних товаров
- Три уровня: точное совпадение → подстрока → токенное (≥50% общих слов)
- `_tokenize_name()`: убирает стоп-слова, нормализует
- `_token_match_score(a, b)`: Jaccard-подобный скор
- Пример: «Вода питьевая 0,5 б/г» ↔ «Вода Бон.Аква б/г 0,5» = 0.67 ✓

### 7.5 Постоянные расходы
- Пользователь вводит ₽/мес: ФОТ, аренда, коммунальные, маркетинг, прочие
- Хранятся в SQLite (`goai_settings.db`)
- `_fixed_cost_for_period()`: полные месяцы = полная сумма, неполные = пропорция дней
- Февраль 28 дней × 21М = ровно 21М (не 19.6М как при делении на 30)

### 7.6 Период-селектор
- Компактный: `[◀][Период dropdown][▶][↻]`
- Пресеты: Сегодня, Вчера, 7/30/90 дней, Произвольный
- Стрелки ◀/▶ сдвигают на 1 день, переключают на «Произвольный»
- **Desktop:** CSS `max-width: 320px` на контейнере
- **Mobile:** JS MutationObserver перебивает inline-стили Streamlit

### 7.7 Прогноз на завтра
- Загружает тот же день недели за последние 8 недель
- Взвешенное среднее (новые недели весят больше)
- Тренд = сравнение последних 2 vs предыдущих 2 недель
- Мини-график истории дня недели
- Контекст передаётся в Gemini для ИИ-рекомендаций

### 7.8 SH API (StoreHouse)
- REST API на `saturn.carbis.ru:7477/api`
- Основные endpoints: `/api/sh5exec` (выполнение процедур)
- **Работающие процедуры:**
  - `GoodsTree` — дерево товаров
  - `GDoc0ExtList` — список приходных накладных
  - `GDoc0` — детали накладной (товары, цены)
  - `GDoc12` — акты нарезки (рецептуры, себестоимость)
  - `FindLinksToCmp` — связи товар → комплекты
  - `GRemns` — остатки по складам
  - `GDoc1_5LstDocs` — список документов по типу
  - `GDoc4` — перемещения между складами
- **НЕ работающие:** GWriteOff, GWaste, GActSpis (списания), Price0 (цены в карточке пусты)

## 10. Известные баги и edge cases

1. **KeyError при sort_values** — если колонка переименована через `t()`, а `sort_values` ссылается на старое имя. Нужно использовать переменную: `_col = t("revenue") + " ₽"; df.sort_values(_col)`
2. **Selectbox IndexError** — при переключении Произвольный ↔ Пресет менялось количество колонок. Решено: `st.rerun()` при смене + разные layouts
3. **SH API timeout** — некоторые процедуры (GRemns) таймаутят на больших складах. Есть `try/except`
4. **Демо-режим** — SQLite не поддерживает все SQL Server функции (DATEPART, DATEADD). Fallback-запросы с `strftime`

## 11. Как продолжать разработку

### Запуск локально
```bash
pip install -r requirements.txt
# Переименовать env → .env если нужно
streamlit run dashboard.py
```

### Правила при добавлении нового текста
1. Добавить ключ в `_TRANSLATIONS["ru"]` и `_TRANSLATIONS["en"]`
2. В коде использовать `t("key")` вместо строки напрямую
3. Заголовки страниц через `tp()`, группы через `tg()`
4. Plotly labels через `_labels()` (функция, не dict)
5. Если dataframe column переименовывается — использовать переменную для sort/filter

### Навигация по коду
- `Ctrl+F "if page =="` — найти начало любой страницы
- `Ctrl+F "def "` — найти функцию
- `Ctrl+F "_TRANSLATIONS"` — словарь переводов (строки ~190–420)
- `Ctrl+F "CHART_THEME"` — настройки графиков (строки ~4380)
- `Ctrl+F "period_selector"` — селектор периода (строки ~2004)
- `Ctrl+F "MENU_GROUPS"` — структура навигации (строки ~4369)

### Деплой
```bash
# Собрать ZIP
zip goai-rest.zip dashboard.py config.toml env env.example gitignore requirements.txt README.md demo_data.json.gz
# Или просто заменить dashboard.py на сервере
```
